# Zsh and Powerlevel10k

**Zsh** is the interactive shell; **Oh My Zsh** is a plugin/theme framework on top
of it; **Powerlevel10k** (P10k) is a fast, richly-segmented prompt theme. Together
they make a shell that's informative (git state, exit codes, contexts) without
feeling slow.

> [!key-insight]
> The thing that makes a "fancy prompt" actually usable is P10k's **instant
> prompt**: a cached prompt is drawn *before* the rest of `.zshrc` finishes
> sourcing, so the shell is interactive immediately even with heavy plugins. The
> instant-prompt block must sit at the very top of `.zshrc`.

## The layers

| Layer | Role |
|-------|------|
| **Zsh** | The shell itself (completion, globbing, history) |
| **Oh My Zsh** | Plugin + theme loader, sane defaults |
| **Powerlevel10k** | The prompt theme (segments, instant prompt) |
| `~/.p10k.zsh` | P10k's generated config (run `p10k configure` to rebuild) |
| `vivid` | Generates `LS_COLORS` from a theme for colored `ls`/`exa` |

A Starship config can be kept side-by-side for reference, but only one prompt
engine runs at a time — if P10k is active, the Starship init line stays commented.

## Config layout

A maintainable setup keeps concerns split rather than one giant `.zshrc`:

```bash
~/.zshrc          # instant prompt, OMZ + plugins, env, tool init, sources ~/.zshrc.d/*
~/.p10k.zsh       # prompt segments/colors (generated by `p10k configure`)
~/.zshrc.d/       # per-domain alias files, all auto-sourced
  git-aliases.zsh
  docker.zsh
  system.zsh      # systemd / journal / boot helpers
  ...
```

Every `*.zsh` in `~/.zshrc.d/` is sourced in a loop, so adding a new alias group
is just dropping a file — no edits to the main `.zshrc`.

## Useful plugins

Built on Oh My Zsh, the high-value plugins:

| Plugin | Purpose |
|--------|---------|
| `git` | Git aliases + prompt info |
| `sudo` | Press `Esc` twice → prefix last command with `sudo` |
| `zsh-autosuggestions` | Fish-style history suggestions (accept with `→`) |
| `zsh-syntax-highlighting` | Inline command highlighting as you type |
| `zsh-completions` | Extra completion definitions |
| `zsh-history-substring-search` | Substring history search with `↑`/`↓` |
| `colored-man-pages` | Colorized `man` |

> [!note]
> Load order matters: `zsh-syntax-highlighting` should be sourced **last**, and
> `zsh-history-substring-search` **after** it, or key bindings/highlighting break.

## Tool integrations

Modern shell quality-of-life, initialized in `.zshrc`:

- **zoxide** — frecency-based `cd` (`z <partial>` jumps to the right dir)
- **fzf** — fuzzy finder (history, files, `Ctrl+R`)
- **direnv** — per-directory env loading (`.envrc`)
- **nvm / pyenv** — Node / Python version management
- **vivid** — `LS_COLORS` from a named theme so `ls`/`exa` match the terminal palette

## Theming it together

The reason it looks cohesive: one palette drives the [[Ghostty Terminal]] colors,
the P10k segment colors, the `vivid` `LS_COLORS`, [[tmux Cheatsheet|tmux]], and the
[[Neovim Cheatsheet|Neovim]] theme. Powerlevel10k needs a **Nerd Font** for its
segment glyphs (powerline separators, git/branch icons) — set that in the terminal.

## Bootstrapping a new machine

A bootstrap script can provision the baseline from scratch — install zsh + modern
CLI tools (zoxide, fzf, ripgrep, fd, bat, exa, neovim), Nerd Fonts, Oh My Zsh and
the plugins, then `chsh` to zsh. The maintained P10k config is then symlinked/copied
in over the minimal starter `.zshrc` the bootstrap writes.
