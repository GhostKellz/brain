# ctl365

An enterprise CLI for automated [[Microsoft 365 Administration]] — deployment,
policy management, and compliance across Intune, Exchange, and SharePoint for
multi-tenant MSP environments. By [[GhostKellz]].

> [!key-insight]
> ctl365 favors safety by default: Conditional Access ships report-only first,
> and operations support dry-run so changes can be previewed before they touch a
> tenant.

## Stack

- [[Rust]] (2024 edition) with tokio
- clap 4.5 CLI and a ratatui TUI
- reqwest with OAuth2; serde for JSON / TOML
- Microsoft Graph v1.0 + beta

## Capabilities

- Baseline deployment (Settings Catalog; OpenIntuneBaseline / CIS-aligned templates; BitLocker / Defender)
- Conditional Access policy packs (report-only first)
- App management (Win32 / M365 Apps / Store / iOS VPP / Android / web)
- Windows Autopilot: bulk CSV import, profiles, sync
- Cross-tenant export / import migration with group mapping
- Audit and drift detection with auto-remediation, compliance scoring, and HTML / CSV / JSON reports
- SharePoint and Viva management
- Copilot / agent discovery; CISA SCuBA audits
- Multi-tenant MSP TUI
- OAuth2 device-code and client-credentials flows
- Dry-run safety
